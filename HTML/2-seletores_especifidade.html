Análise:

  • Regra 1: é bem genérica (li) e quase sempre é sobrescrita por outras mais específicas.
  • Regra 2 e 3: possuem a mesma especificidade (0,0,1).
  • Regra 4: combina seletor adjacente com classe, ganhando prioridade contextual.
  • Regra 5: inclui um seletor de atributo com classe → especificidade maior.
  • Regra 6: inclui ID, que sempre vence classes e tags simples.
  • Regra 7: combina ID + classe, vencendo qualquer regra anterior.
  • Regra 8: um seletor super específico com tag > classe + ID — tem enorme força, mas só se aplica ao item com id="lip5", que não existe no código atual.
